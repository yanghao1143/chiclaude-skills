## 📖 技能简介

系统化的提示工程技术，包括角色设定、少样本学习、思维链、结构化输出和提示优化策略。适用于设计有效的 AI 提示、优化 LLM 输出质量。

## 🎭 核心模式

### 1. 角色设定 (Role Playing)

```
你是一位经验丰富的 Python 开发专家，专注于代码质量和性能优化。
你有 15 年的软件开发经验，精通设计模式和最佳实践。

请审查以下代码并提供改进建议：
```

### 2. 少样本学习 (Few-Shot Learning)

```
将以下句子转换为正式的商务邮件：

示例 1:
输入: 会议改到明天
输出: 尊敬的各位，由于日程调整，我们的会议将改期至明天举行。感谢您的理解。

示例 2:
输入: 项目延期了
输出: 尊敬的客户，由于技术原因，项目交付时间需要相应调整。我们深表歉意。

现在请转换:
输入: 需要增加预算
```

### 3. 思维链 (Chain of Thought)

```
请一步步思考以下问题：

问题：一个农场有鸡和兔子共 35 只头，94 只脚，问鸡和兔子各多少只？

思考过程：
1. 设鸡有 x 只，兔子有 y 只
2. 根据头数：x + y = 35
3. 根据脚数：2x + 4y = 94
4. 从第一个方程：x = 35 - y
5. 代入第二个方程：2(35-y) + 4y = 94
6. 展开：70 - 2y + 4y = 94
7. 简化：2y = 24
8. 解得：y = 12（兔子），x = 23（鸡）

答案：鸡 23 只，兔子 12 只
```

### 4. 结构化输出

```
请分析以下文本的情感，以 JSON 格式输出：

文本：这款产品真的很好用，物流也很快，但是价格稍微有点贵。

输出格式：
{
  "overall_sentiment": "positive/neutral/negative",
  "sentiment_score": 0.0-1.0,
  "aspects": [
    {
      "aspect": "产品",
      "sentiment": "positive",
      "keywords": ["好用"]
    }
  ]
}
```

## 📝 提示优化策略

### 清晰性原则

```
# 不好的提示
写一个函数

# 好的提示
请用 Python 编写一个函数，功能如下：
- 函数名：calculate_average
- 参数：一个数字列表
- 返回值：列表中所有数字的平均值（浮点数）
- 如果列表为空，返回 None
- 添加类型提示和文档字符串
```

### 分步指令

```
请按以下步骤完成任务：

第 1 步：阅读并理解需求
[需求内容]

第 2 步：设计解决方案
[输出设计方案]

第 3 步：实现代码
[输出代码]

第 4 步：编写测试用例
[输出测试]
```

### 约束条件

```
请编写代码，遵循以下约束：

约束：
1. 代码必须兼容 Python 3.9+
2. 不使用任何第三方库
3. 时间复杂度不超过 O(n log n)
4. 包含完整的错误处理
5. 添加中文注释
```

## 🔧 高级技术

### 自我修正

```
请先给出答案，然后检查你的答案是否正确：

问题：[问题内容]

你的答案：[答案]

自我检查：
- 答案是否完整？
- 是否有遗漏的边界情况？
- 格式是否正确？

修正后的答案：[最终答案]
```

### 提示组合

```
# 角色 + 任务 + 格式 + 约束
你是一位专业的技术文档撰写专家。[角色]

请为以下 API 编写文档：[任务]

文档格式：
## 概述
## 参数
## 返回值
## 示例
## 注意事项

要求：
- 使用中文
- 每个参数包含类型和默认值
- 提供至少 2 个使用示例
```

### 迭代优化

```
我对上一次的输出不满意，请按以下方式改进：

原输出：[之前的输出]

改进要求：
1. 增加更多细节
2. 使用更专业术语
3. 添加代码注释

请重新生成：
```
