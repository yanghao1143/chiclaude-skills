## 🎨 设计指南

**不要创建无聊的幻灯片。** 纯粹的项目符号和白背景不会给任何人留下深刻印象。

### 开始之前

- **选择大胆、内容相关的配色方案**：配色应该为此主题专门设计
- **主导而非平等**：一种颜色应占主导（60-70% 视觉权重），配以 1-2 种支持色和一种强调色
- **深浅对比**：深色背景用于标题+结尾幻灯片，浅色用于内容（"三明治"结构）
- **坚持视觉主题**：选择一个独特元素并在每张幻灯片上重复

### 配色方案

| 主题 | 主色 | 辅色 | 强调色 |
|------|------|------|--------|
| **午夜高管** | `1E2761` (海军蓝) | `CADCFC` (冰蓝) | `FFFFFF` (白) |
| **森林与苔藓** | `2C5F2D` (森林绿) | `97BC62` (苔藓绿) | `F5F5F5` (奶油色) |
| **珊瑚活力** | `F96167` (珊瑚红) | `F9E795` (金色) | `2F3C7E` (海军蓝) |
| **温暖陶土** | `B85042` (陶土红) | `E7E8D1` (沙色) | `A7BEAE` (鼠尾草绿) |
| **海洋渐变** | `065A82` (深蓝) | `1C7293` (青色) | `21295C` (午夜蓝) |
| **炭灰极简** | `36454F` (炭灰) | `F2F2F2` (灰白) | `212121` (黑) |

### 每张幻灯片

**每张幻灯片都需要视觉元素** — 图片、图表、图标或形状。纯文字幻灯片容易被遗忘。

**布局选项：**
- 两栏（左文字，右插图）
- 图标+文字行（彩色圆圈中的图标，粗体标题，下方描述）
- 2x2 或 2x3 网格
- 半出血图片（左侧或右侧全出血）配内容叠加

**数据展示：**
- 大数字标注（60-72pt 大数字，下方小标签）
- 对比列（前后对比、优缺点、并排选项）
- 时间线或流程图（编号步骤，箭头）

### 排版

**选择有趣的字体搭配** — 不要默认使用 Arial。

| 标题字体 | 正文字体 |
|----------|----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |

| 元素 | 大小 |
|------|------|
| 幻灯片标题 | 36-44pt 粗体 |
| 章节标题 | 20-24pt 粗体 |
| 正文文本 | 14-16pt |
| 说明文字 | 10-12pt 浅色 |

### 间距

- 最小边距 0.5 英寸
- 内容块之间 0.3-0.5 英寸
- 留出呼吸空间 — 不要填满每一寸

### 避免（常见错误）

- **不要重复相同布局** — 在幻灯片之间变化使用列、卡片和标注
- **不要居中正文** — 段落和列表左对齐；只居中标题
- **不要吝啬大小对比** — 标题需要 36pt+ 才能与 14-16pt 正文区分
- **不要默认使用蓝色** — 选择反映特定主题的颜色
- **不要创建纯文字幻灯片** — 添加图片、图标、图表或视觉元素

## ✅ QA 检查（必需）

**假设有问题。你的工作是找到它们。**

### 内容 QA

```bash
python -m markitdown output.pptx
```

检查缺失内容、拼写错误、错误顺序。

### 视觉 QA

将幻灯片转换为图像后检查：

- 重叠元素
- 文本溢出或在边缘/边界被截断
- 元素太近（< 0.3" 间距）
- 间距不均匀
- 边距不足（< 0.5"）
- 列或类似元素未对齐
- 低对比度文本
- 残留占位符内容
